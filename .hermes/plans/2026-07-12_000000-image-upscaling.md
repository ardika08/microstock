# Plan: Image Upscaling Feature — autofillstock.my.id

**Date:** 2026-07-12  
**Status:** Approved — ready for implementation

---

## Goal

Tambah fitur **Image Upscaling** ke dashboard autofillstock.my.id dengan 2 tier kualitas.  
User upload gambar → backend kirim ke Replicate API → user download hasil.  
Dikonsumsi **kredit** seperti generate metadata.

## 2 Tier Upscaling

| Tier | Model Replicate | Kualitas | Harga/run | Kredit |
|---|---|---|---|---|
| **Standard** | `nightmareai/real-esrgan` | ⭐⭐⭐⭐ | ~$0.002 | **2 kredit** |

## Plan Implementasi

### Task 1 — Backend API endpoint
**File baru:** `pages/api/upscale.ts`

```typescript
// POST /api/upscale
// Body: { imageBase64: string, filename: string, tier: 'standard' | 'premium' }
// Response: { upscaledImageBase64: string, creditsRemaining: number }

export const config = { api: { bodyParser: { sizeLimit: '10mb' } } }
```

Flow:
1. Auth check (`getServerSession`)
2. Validasi `tier` — standard atau premium
3. Cek credits: standard >= 2, premium >= 5
4. Kirim ke Replicate API:
   - Standard: `nightmareai/real-esrgan` dengan `scale: 4`
   - Premium: `philz1337x/clarity-upscaler` dengan `scale_factor: 2`
5. Poll hasil dari Replicate (async job) — max 60 detik
6. Deduct kredit dari DB (2 atau 5)
7. Log ke `generate_history` (platform: "web", type: "upscale_standard" / "upscale_premium")
8. Return hasil sebagai base64 ke frontend

## Replicate API Flow

```
Frontend (base64 image)
    ↓
POST /api/upscale
    ↓
POST https://api.replicate.com/v1/predictions
{
  "version": "<model_version_id>",
  "input": { "image": "data:image/jpeg;base64,...", "scale": 4 }
}
    ↓
GET https://api.replicate.com/v1/predictions/{id}  (polling setiap 2 detik)
    ↓
Hasil: URL gambar dari Replicate CDN
    ↓
Fetch URL → convert ke base64 → return ke frontend
```

## Risiko & Mitigasi

| Risiko | Mitigasi |
|---|---|
| Replicate API timeout | Max poll 60 detik, kredit tidak dipotong jika gagal |
| File > 5MB | Validasi frontend + backend guard |
| Premium model lambat | Loading state yang informatif ("Sedang memproses...") |
| Kredit tidak cukup | Cek sebelum call API, error message jelas |
