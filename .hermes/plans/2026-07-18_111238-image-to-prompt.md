# Image-to-Prompt Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.  
> **Status:** PLAN ONLY — do not implement until user approves.

**Goal:** Tambah fitur **Image → Prompt** di dashboard Autofillstock: user upload gambar, AI (OpenAI Vision) menghasilkan prompt text siap pakai, **1 generate = 1 kredit**, sistem API key sama seperti Generate Metadata.

**Architecture:** Reuse pola `/api/generate` + halaman dashboard (mirip Generate / Upscale). Frontend upload image (base64) → API server-side cek session + kredit/API key → panggil OpenAI Vision (`gpt-4o` / `gpt-4o-mini`) → return prompt (+ optional variants) → potong 1 kredit + log history. **Tidak** pakai Replicate/Fal. Seamless Pattern tetap di-skip / Coming Soon.

**Tech Stack:** Next.js pages router, NextAuth session, Neon PG + Drizzle, OpenAI Chat Completions Vision, existing credit model (`users.credits` / lifetime own key).

## Product decisions (locked for MVP)

| Item | Decision |
|---|---|
| Input | **Image only** (JPG/PNG/WebP), max ~5–8MB body (sudah ada limit 8mb di generate) |
| Output | **1 primary prompt** (English, detailed). Optional: 2 short variants di v1.1 |
| Credit | **1 generate = 1 kredit** (free/topup). Lifetime = own OpenAI key, no credit cut. Starter = fair-use track (sama generate metadata) |
| API key | Sama rules `/api/generate`: free/topup/starter → server `OPENAI_API_KEY`; lifetime → user key |
| Model | Vision: `gpt-4o` (atau `gpt-4o-mini` kalau mau hemat — default **gpt-4o** biar konsisten metadata) |
| Placement | Halaman baru `/dashboard/prompt` + menu sidebar **Image Prompt** |
| Seamless | **Skip** — jangan kerjakan backend seamless di plan ini |
| Extension | **Tidak** di MVP (dashboard only) |

## API design

### `POST /api/prompt`

**Auth:** NextAuth session required.

**Body:**
```json
{
  "imageBase64": "data:image/jpeg;base64,...",
  "style": "general" | "midjourney" | "flux" | "sdxl",
  "userApiKey": "sk-..." // optional, lifetime
}
```

**Success 200:**
```json
{
  "success": true,
  "prompt": "A highly detailed ...",
  "negativePrompt": "blurry, low quality, ...",  // optional, include in MVP if cheap
  "creditsRemaining": 12
}
```

**Errors:** 401 login, 400 no image / invalid type, 402 kredit habis, 429 fair use, 500 OpenAI fail.

**OpenAI system prompt (MVP):**
- Role: expert image-to-prompt engineer for generative AI / microstock workflow
- Output **strict JSON only**: `{ "prompt": string, "negativePrompt": string, "tags": string[] }`
- Rules: English, concrete visual details (subject, composition, lighting, style, camera), no markdown, prompt 40–120 words, no copyrighted character/brand names

**Model call:** same vision structure as generate.ts:
```ts
content: [
  { type: 'text', text: instruction },
  { type: 'image_url', image_url: { url: imageBase64, detail: 'low' } }
]
```

### Task 2: Create `POST /api/prompt`

**Objective:** Endpoint image→prompt + credit rules.

**Files:**
- Create: `pages/api/prompt.ts`

**Implementation checklist:**
1. `export const config = { api: { bodyParser: { sizeLimit: '8mb' } } }`
2. Method POST only
3. `getServerSession` + load user from Neon (copy generate.ts pattern)
4. Validate `imageBase64` starts with `data:image/`
5. Resolve apiKey + model (copy free/topup/starter/lifetime logic from `pages/api/generate.ts`)
6. Call OpenAI vision with JSON response_format
7. Parse `{ prompt, negativePrompt?, tags? }`
8. Deduct 1 credit for free/topup; starter creditsUsed++; lifetime no deduct
9. Insert `generateHistory` with `platform: 'image_prompt'`, `filename: 'image-prompt'`, `title: prompt.slice(0, 180)`, `creditsUsed: 1`
10. Return `{ success, prompt, negativePrompt, creditsRemaining }`
11. On OpenAI failure: **do not** deduct credits (deduct after success only — same order as generate.ts currently does after success ✓)

**Verify:**
```bash
# unauth
curl -s -o /dev/null -w "%{http_code}" -X POST https://autofillstock.my.id/api/prompt
# expect 401
```

**Commit:** `feat: add /api/prompt image-to-prompt endpoint`

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| Base64 body too large / 413 | Client compress/resize to max edge 1280px before upload; keep 8mb limit |
| Credit deducted on OpenAI fail | Deduct **after** successful parse only |
| Lifetime key vs server key mixup | Copy exact branch from `generate.ts` |
| Cost of vision higher than text | `detail: 'low'`; monitor; optional switch mini later |
| Prompt contains brand/character | System instruction forbid; user still responsible |
| History table semantics odd for prompts | Use platform flag + truncated title |
